CiteClaw UI — design polish project

CiteClaw is a literature-discovery workspace: users assemble a screening pipeline (Build), watch it stream papers into a live citation graph (Run), and explore the finished collection (Explore). This project contains the real production UI, captured from the running app.

The one rule: redesign anything visual — markup, layout, structure, styling — but the frontend must keep speaking to the current backend exactly as today. The full frozen API surface and the affordances that must survive are in docs/api-contract.md. Engineering re-wires approved designs and gates compatibility.

What's here

  • Foundations — the real design tokens (--cc-*) and type scale, parsed from the production stylesheet.
  • Components — live-captured markup per component, grouped by area (Chrome / Build / Run / Graph / Explore / Overlays). Each renders against assets/app.css, the actual stylesheet.
  • Screens — full-page screenshots for composition context.
  • src/jsx/ — the actual React sources behind every component.

How to propose changes

  • Prefer editing assets/app.css (tokens first — they restyle the whole app) and the component cards' markup in place.
  • The app ships the data-palette="mono" light variant; keep colors going through tokens so the dark theme keeps working.
  • Structural redesigns are welcome — annotate what moved and why, so re-wiring the React components is unambiguous.
